# 面向少样本医学图像分割的 SAM 改进方法研究

---

## 零、方案可行性分析

### 0.1 技术可行性 —— 高

**（1）SAM 模型生态成熟**
- Segment Anything Model (SAM) 由 Meta 于 2023 年开源，在 HuggingFace、GitHub 均有完整实现，`pip install segment-anything` 即可使用。
- SAM 提供三种规模：ViT-H (636M)、ViT-L (308M)、ViT-B (91M)。**建议选用 ViT-B**，在单张 12GB 显存 GPU（如 RTX 3060/4060、T4、A10）上即可完成推理和轻量微调，适合课程作业场景。

**（2）医学图像数据集公开可得**
- 推荐使用 **2D 数据集**（不要用 3D CT/MRI，计算量过大）：
  - **Kvasir-SEG**（息肉分割）：1000 张，标注完整，二分类分割，难度适中
  - **ISIC 2018**（皮肤病灶分割）：2594 张，标注完整，类别丰富
  - **BUSI**（乳腺超声分割）：780 张，含正常/良性/恶性三类
  - **COVID-19 CT Lung Segmentation**：CT 切片，前景背景分割
- 建议优先选 **Kvasir-SEG 或 ISIC 2018**，数据量适中、任务清晰、论文中常见，对比基线好找。

**（3）改进方法有明确学术支撑**
- 特征适配（Adapter/LoRA）：2023-2025 年有大量论文验证了在 SAM 上加 Adapter 或 LoRA 的有效性（如 SAMed、SAM-Adapter、Medical SAM Adapter 等）。
- 提示优化：自动提示生成（如基于粗分割结果的网格采样、中心点采样）是成熟策略。
- 解码器改进：U-Net 风格解码器 + 跳跃连接在医学图像中已被广泛验证。

**（4）代码实现门槛可控**
- SAM 官方代码结构清晰，图像编码器 → 提示编码器 → 掩码解码器三部分独立，修改局部模块不影响整体。
- 主流深度学习框架（PyTorch + transformers + monai）都有现成的医学图像处理工具。
- LoRA 可通过 `peft` 库几行代码实现，Adapter 可用简单的 `nn.Linear` 降维-升维结构。

### 0.2 资源可行性 —— 高

- ViT-B SAM 在 batch_size=1 时约需 6-8GB 显存做推理，加上 LoRA 微调约需 10-12GB。

- 如果显存不足，可进一步缩小输入尺寸（1024→512），或冻结图像编码器。

### 0.3 方案可行性 —— 中等偏高

**优势：**
- 选题贴合医学图像 + 大模型微调的前沿方向，创新性足够。
- 方法模块化设计，每个模块可独立开发、独立验证，降低耦合度。
- 即使某个模块效果不理想，消融实验本身就能作为分析内容写入论文。
- SAM + Adapter/LoRA 范式有充足的参考文献支撑，"不像凭空编造"。

**主要风险与应对策略：**

| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|----------|
| SAM ViT-B 显存不足 | 中 | 高 | 换 ViT-Tiny 或缩小输入分辨率至 512×512 |
| 微调后效果不如原始 SAM | 中 | 中 | 重点分析"为什么"，写入讨论；强调少样本条件下的相对提升 |
| 提示优化实现困难 | 中 | 低 | 降级为"固定网格点提示"，作为 baseline 策略 |
| 算力不够跑完整实验 | 低 | 高 | 使用 Colab / AutoDL，或减少训练轮数做概念验证 |
| 数据集下载/预处理出问题 | 低 | 中 | 提前下载并备份到团队共享网盘 |

### 0.4 对比：为什么不用更简单的方案？

| 方案 | 优点 | 缺点 | 结论 |
|------|------|------|------|
| 纯 U-Net 医学分割 | 简单成熟 | 缺乏创新性，课程大作业分数受限 | ❌ |
| SAM + 全量微调 | 效果好 | 计算量大，7人团队都在等训练，分工不均衡 | ❌ |
| SAM + Adapter/LoRA（本方案）| 创新性好，模块化强，每人可独立工作 | 需要协调接口 | ✅ |

### 0.5 可行性结论

**该方案技术上可行、资源上可支撑、风险可控。** 建议按照"先跑通 baseline → 逐个加模块 → 消融验证"的路径推进，确保始终有一个可工作的版本。

## 一、补充细节：具体技术方案

### 1.1 SAM 主干网络具体配置

```
选择：SAM ViT-B (sam_vit_b_01ec64.pth)
- 图像编码器：ViT-Base，12层 Transformer，768维隐藏层
- 提示编码器：位置编码 + CLIP 文本编码（本次不使用文本提示）
- 掩码解码器：两层 Transformer 解码器 + 上采样头
- 输入分辨率：1024×1024（可降至 512×512 节省显存）
- 冻结策略：冻结图像编码器前 9 层，仅微调后 3 层 + 所有 Adapter 层
```

### 1.2 特征适配模块 —— LoRA 方案（推荐）

**为什么选 LoRA 而不是 Adapter？**
- LoRA 不增加推理延迟（可合并到原权重）
- PEFT 库原生支持，代码量极少
- 参数量可控（rank=4 时仅增加约 0.1% 参数）

**具体配置：**
```
- 目标层：图像编码器所有 Attention 层的 Q、V 投影矩阵
- LoRA rank：r = 4（或 8，视显存而定）
- LoRA alpha：16
- LoRA dropout：0.1
- 实现：使用 huggingface/peft 库的 LoraConfig
```

**替代方案 —— Adapter：**
如果不用 LoRA，可在每个 Transformer 块的 FFN 后插入 Adapter：
```
Adapter(x) = x + Linear_up(GELU(Linear_down(x)))
Linear_down: d_model → d_bottleneck (如 768→64)
Linear_up:   d_bottleneck → d_model (如 64→768)
```

### 1.3 提示优化模块 —— 具体策略

**方案 A（推荐，实现简单）：网格点采样 + 置信度筛选**
1. 在图像上生成 32×32 的均匀网格点
2. 每个点作为正向提示输入 SAM
3. SAM 输出每个点的 IoU 预测值
4. 选择 IoU > 0.5 的点作为最终提示集合
5. 对这些点生成的掩码做 NMS + 融合

**方案 B（效果更好，实现稍难）：基于粗分割的提示生成**
1. 先用轻量模型（如 MobileNet + 小 U-Net）生成粗分割
2. 从粗分割结果中采样：在目标区域内部随机采样正点，背景区域采样负点
3. 正负点一起输入 SAM 的提示编码器

**建议：先用方案 A，时间充裕再尝试方案 B。**

### 1.4 医学分割解码模块 —— U-Net 风格跳跃连接

```
设计：
- 从 SAM 图像编码器提取 4 个中间层特征（layer 3, 6, 9, 12）
- 每层特征经过 1×1 卷积降维到统一通道数（如 64）
- 上采样路径：深特征 → 上采样 → concat(浅特征) → ConvBlock
- 最终输出：单通道分割掩码（sigmoid）或多通道分类掩码（softmax）
- 与 SAM 原始掩码解码器输出加权融合：final = α * SAM_mask + (1-α) * MedDecoder_mask
```

### 1.5 损失函数 —— 推荐组合

```python
# 推荐组合：DiceLoss + BCEWithLogitsLoss
class CombinedLoss(nn.Module):
    def __init__(self, dice_weight=0.5, bce_weight=0.5):
        self.dice_loss = DiceLoss()       # 1 - 2*|pred∩gt|/(|pred|+|gt|)
        self.bce_loss = nn.BCEWithLogitsLoss()

    def forward(self, pred, target):
        return (self.dice_weight * self.dice_loss(pred, target) +
                self.bce_weight * self.bce_loss(pred, target))
```

### 1.6 少样本场景构造方法

为了模拟"少样本"场景，从完整训练集中按比例采样：
```
- 1% 少样本：~10-20 张训练图像
- 5% 少样本：~50-100 张训练图像
- 10% 少样本：~100-200 张训练图像
- 100% 全量：完整训练集（作为上界对比）
```
注意：验证集和测试集保持完整，不做采样。

## 二、补充细节：实验设计

### 2.1 推荐数据集细节

**首选：Kvasir-SEG（息肉分割）**
```
- 来源：https://datasets.simula.no/kvasir-seg/
- 图像数量：1000 张（分辨率从 332×487 到 1920×1072 不等）
- 标注：像素级二值掩码（息肉=1，背景=0）
- 划分：800 训练 / 100 验证 / 100 测试
- 预处理：统一 resize 到 1024×1024，像素归一化到 [0,1]
```

**备选：ISIC 2018（皮肤病灶分割）**
```
- 来源：https://challenge.isic-archive.com/data/
- 图像数量：2594 张 + 对应标注
- 标注：像素级二值掩码（病灶=1，背景=0）
- 划分：2000 训练 / 294 验证 / 300 测试
```

### 2.2 训练超参数建议

```
- 优化器：AdamW (lr=1e-4, weight_decay=1e-4)
- 学习率调度：CosineAnnealingLR + 5 epoch warmup
- Batch size：2-4（1024 分辨率）/ 8-16（512 分辨率）
- 训练轮数：50-100 epochs（少样本时更多轮数）
- 数据增强：随机水平翻转、随机旋转(±10°)、随机亮度对比度、弹性变形
- 早停：验证 Dice 连续 10 轮不提升则停止
```

### 2.3 对比基线清单

| 基线方法 | 说明 | 实现难度 |
|----------|------|----------|
| 原始 SAM（网格提示）| 不做任何微调，使用网格点提示 | ⭐ |
| U-Net（从头训练）| 经典医学分割基线 | ⭐ |
| U-Net（ImageNet 预训练编码器）| 迁移学习版本 | ⭐⭐ |
| SAM + 全量微调（仅解码器）| 冻结编码器，训练掩码解码器 | ⭐⭐ |
| SAM + LoRA（本方案-仅适配）| 加 LoRA 但不加医学解码器 | ⭐⭐ |
| **SAM + LoRA + MedDecoder（完整方案）** | 特征适配 + 医学解码器 | ⭐⭐⭐ |

### 2.4 消融实验设计

```
实验组设计：
1. 原始 SAM                         → 基准
2. SAM + LoRA only                  → 验证特征适配的贡献
3. SAM + MedDecoder only            → 验证医学解码器的贡献  
4. SAM + LoRA + MedDecoder          → 完整方案
5. 完整方案 on 1%/5%/10%/100% data  → 验证少样本性能

每个实验组记录：Dice, IoU, Precision, Recall
```

## 五、已知局限与未来可扩展方向

### 局限（论文讨论部分可用）
1. 仅验证了 2D 医学图像，未测试 3D 体积数据（CT/MRI）
2. 提示策略仍依赖网格采样，未考虑病理学先验知识

1. 引入 CLIP/SigLIP 文本编码器，实现文本驱动的零样本分割
2. 扩展到 3D 医学体积数据（3D SAM + 3D Adapter）
3. 结合主动学习（active learning）策略优化样本选择
