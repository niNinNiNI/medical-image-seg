# 项目实施计划：面向少样本医学图像分割的 SAM 改进方法研究

---

## 一、项目概述与目标

### 1.1 项目背景

本项目旨在改进 Meta 的 Segment Anything Model (SAM)，使其适用于少样本医学图像分割场景。核心思路是：在冻结 SAM 大部分参数的前提下，通过引入 **LoRA 特征适配**、**自动提示优化**和 **医学跳跃连接解码器** 三个模块，以极低的微调成本（参数量增加约 0.1%）实现 SAM 在医学图像上的高效迁移。

### 1.2 核心目标

| 目标 | 衡量标准 |
|------|----------|
| SAM baseline 在选定医学数据集上可运行 | 推理脚本无报错，输出合法分割掩码 |
| LoRA 微调后性能提升 | 少样本条件下 Dice 比 baseline 提升 ≥5% |
| 完整方案（LoRA + MedDecoder）达到最优 | 在 1%/5%/10% 少样本场景下 Dice 最高 |
| 消融实验完整 | 至少 5 组实验数据（含 baseline 对比） |
| 论文可达发表水平 | 图表完整、论证充分、参考文献 ≥15 篇 |

## 四、阶段二：核心模块实现

### 4.1 SAM 主干加载模块（`models/sam_backbone.py`）

**功能描述：** 加载预训练 SAM ViT-B 模型，提供冻结/解冻接口，暴露中间层特征。

**实现要点：**

1. 使用 `segment_anything.sam_model_registry["vit_b"]` 加载官方权重 `sam_vit_b_01ec64.pth`
2. 实现分层冻结策略：
   - 默认：冻结图像编码器前 9 层 Transformer
   - 后 3 层 + 掩码解码器可训练
   - 提示编码器固定冻结
3. 注册 forward hook 提取 4 个中间层特征（layer 3, 6, 9, 12），供医学解码器使用
4. 提供 `train(mode)` 覆盖方法，确保冻结层始终处于 eval 模式

**接口定义：**

        # }

### 4.2 LoRA 适配模块（`models/lora_adapter.py`）

**功能描述：** 在 SAM 图像编码器的 Attention Q/V 投影矩阵上添加 LoRA 低秩适配层。

**实现要点：**

1. 使用 `peft.LoraConfig` 配置：
   ```python
   LoraConfig(
       r=4,                     # LoRA rank
       lora_alpha=16,           # 缩放因子
       target_modules=["q_proj", "v_proj"],  # 仅适配 Q、V
       lora_dropout=0.1,
       bias="none",
   )
   ```
2. 将 SAM 图像编码器包装为 `get_peft_model(sam.image_encoder, lora_config)`
3. 训练完成后提供 `merge_and_unload()` 方法将 LoRA 权重合并至原模型，消除推理延迟
4. 记录 LoRA 参数量（约原始参数的 0.1%），用于论文中的效率分析

**验证方式：** 对比加载 LoRA 前后模型参数量和前向传播结果的一致性（merge 前 vs merge 后）。

### 4.3 提示优化模块（`models/prompt_optimizer.py`）

**功能描述：** 自动生成高质量的提示点，替代手工标注提示。

**首选方案：网格点采样 + 置信度筛选**

实现步骤：
1. 在图像空间生成 N×N（默认 32×32）的均匀网格点坐标
2. 将每个网格点作为正向提示（label=1）分别输入 SAM
3. SAM 返回每个点的预测掩码及 IoU 预测值
4. 筛选 IoU > 阈值（默认 0.5）的点作为最终提示集合
5. 对筛选后的提示点执行以下后处理：
   - 将多提示合并为单次推理（batch 所有有效提示点）
   - 对各提示生成的掩码执行 NMS（Non-Maximum Suppression）
   - 最终掩码 = 去重后的掩码加权平均（权重 = IoU 预测值）

**备选方案：基于粗分割的提示生成**

若方案 A 效果不理想，替换为：
1. 使用轻量模型（MobileNetV3 + 小型 U-Net）生成粗分割掩码
2. 在粗掩码前景区域内随机采样 5-10 个正点
3. 在粗掩码背景区域内随机采样 5-10 个负点
4. 正负点组合作为 SAM 提示输入

**模块接口：**

### 4.4 医学分割解码器（`models/med_decoder.py`）

**功能描述：** U-Net 风格的跳跃连接解码器，融合 SAM 编码器的多层特征，输出精细的分割掩码。

**架构设计：**

```
输入：SAM 图像编码器的 4 个中间层特征
  feat3:  (B, 768, 64, 64)   ← layer 3
  feat6:  (B, 768, 64, 64)   ← layer 6
  feat9:  (B, 768, 64, 64)   ← layer 9
  feat12: (B, 768, 64, 64)   ← layer 12（最深）

处理流程：
  1. 每层特征经 1×1 卷积降维到 64 通道
  2. feat12 → UpSample(2×) → ConvBlock → concat(feat9) → ConvBlock
     → UpSample(2×) → ConvBlock → concat(feat6) → ConvBlock
     → UpSample(2×) → ConvBlock → concat(feat3) → ConvBlock
     → UpSample(4×) → 恢复到原图分辨率 (1024×1024)
  3. 最终 1×1 卷积输出单通道分割掩码

ConvBlock = Conv2d(3×3) → BatchNorm → ReLU → Conv2d(3×3) → BatchNorm → ReLU
```

**融合策略：**

```python
final_mask = α * SAM_mask_decoder_output + (1 - α) * MedDecoder_output
```
其中 α 为可学习参数（初始化为 0.5），也可设为固定超参数通过消融实验确定。

### 4.5 完整模型整合（`models/full_model.py`）

**功能描述：** 将上述四个模块组合为端到端模型。

```python
class SAMMedSeg(nn.Module):
    def __init__(self, sam_checkpoint, lora_config, prompt_strategy):
        self.backbone = SAMBackbone(sam_checkpoint)
        self.lora_adapter = LoRAAdapter(self.backbone.image_encoder, lora_config)
        self.prompt_optimizer = PromptOptimizer(strategy=prompt_strategy)
        self.med_decoder = MedDecoder()
        self.fusion_alpha = nn.Parameter(torch.tensor(0.5))

    def forward(self, image, gt_mask=None):
        # 1. 编码
        features = self.backbone(image)
        # 2. 提示优化（训练时可用 gt_mask 辅助，推理时自动生成）
        prompts = self.prompt_optimizer(self.backbone, image)
        # 3. SAM 解码路径
        sam_mask = self.backbone.mask_decoder(
            features["image_embeddings"], prompts
        )
        # 4. 医学解码路径
        med_mask = self.med_decoder(features["intermediate_features"])
        # 5. 融合
        final_mask = self.fusion_alpha * sam_mask + (1 - self.fusion_alpha) * med_mask
        return final_mask
```

## 五、阶段三：损失函数、训练与评估

### 5.1 损失函数设计（`losses/combined_loss.py`）

**推荐组合：DiceLoss + BCEWithLogitsLoss（1:1 加权）**

```python
class CombinedLoss(nn.Module):
    def __init__(self, dice_weight=0.5, bce_weight=0.5, smooth=1e-6):
        self.dice_weight = dice_weight
        self.bce_weight = bce_weight
        self.smooth = smooth

    def dice_loss(self, pred, target):
        pred = torch.sigmoid(pred)
        intersection = (pred * target).sum(dim=(2, 3))
        union = pred.sum(dim=(2, 3)) + target.sum(dim=(2, 3))
        dice = (2.0 * intersection + self.smooth) / (union + self.smooth)
        return 1 - dice.mean()

    def forward(self, pred, target):
        return (self.dice_weight * self.dice_loss(pred, target) +
                self.bce_weight * F.binary_cross_entropy_with_logits(pred, target))
```

**备选：** 如分割目标存在类别不均衡（背景 >> 前景），可加入 Focal Loss 或 Tversky Loss。

### 5.2 训练策略与超参数

| 超参数 | 推荐值 | 说明 |
|--------|--------|------|
| 优化器 | AdamW | lr=1e-4, weight_decay=1e-4 |
| 学习率调度 | CosineAnnealingLR | 配合 5 epoch 线性 warmup |
| Batch size | 2-4（1024 分辨率） | 根据显存动态调整 |
| 训练轮数 | 50-100 epochs | 少样本场景适当增加 |
| 早停策略 | patience=10 | 监控验证集 Dice 指标 |
| 混合精度 | AMP (fp16) | 节省显存，加速训练 |
| 梯度裁剪 | max_norm=1.0 | 稳定训练 |

**训练流程（`train.py`）：**

1. 加载配置（`configs/config.yaml`）
2. 初始化模型、损失函数、优化器、学习率调度器
3. 每个 epoch：
   - 训练阶段：前向传播 → 计算损失 → 反向传播 → 参数更新
   - 验证阶段：计算 Dice、IoU，记录最佳模型
   - TensorBoard/WandB 记录损失和指标曲线
4. 早停检查：验证 Dice 连续 10 轮不提升则终止
5. 保存最佳模型权重（基于验证 Dice）

### 6.3 少样本场景构造

```python
def create_few_shot_subset(dataset, ratio):
    """
    从完整训练集中按比例随机采样
    - ratio=0.01: 约 8 张（Kvasir-SEG 800 训练集 × 1%）
    - ratio=0.05: 约 40 张
    - ratio=0.10: 约 80 张
    - ratio=1.00: 全部 800 张
    """
    num_samples = int(len(dataset) * ratio)
    indices = random.sample(range(len(dataset)), num_samples)
    return torch.utils.data.Subset(dataset, indices)
```

**关键约束：**
- 固定随机种子（seed=42）确保可复现
- 验证集和测试集始终保持完整，不做采样
- 每个少样本比例至少跑 3 次（不同随机种子），报告均值 ± 标准差

- Git 提交信息格式：`[模块名] 简要描述`，如 `[models] 实现 LoRA 适配模块`

### 8.3 版本管理

- 使用 GitHub 私有仓库托管代码
- 数据集和模型权重通过 `.gitignore` 排除，使用网盘共享
- 每次实验运行使用 Git tag 标记使用的代码版本

## 九、质量保障与风险应对

### 9.1 模块单元测试

| 测试对象 | 测试内容 |
|----------|----------|
| `MedicalSegDataset` | 数据加载形状正确，少样本采样数量正确 |
| `SAMBackbone` | 加载权重不报错，输出特征维度正确 |
| `LoRAAdapter` | LoRA 参数注入成功，merge 前后输出一致 |
| `PromptOptimizer` | 网格点数量正确，IoU 筛选逻辑正确 |
| `MedDecoder` | 输入输出维度匹配，梯度可正常反传 |
| `CombinedLoss` | 损失值范围合理（0~1），梯度非零 |
| `FullModel` | 端到端前向传播不报错，输出形状正确 |

### 9.2 关键风险与应急方案

| 风险 | 应对措施 |
|------|----------|
| SAM ViT-B 显存不足 | 降至 ViT-Tiny + 512×512 分辨率 |
| LoRA 微调后效果不如原始 SAM | 分析"灾难性遗忘"现象，调整 LoRA rank/layer 选择，写入论文 discussion |
| 提示优化模块难以收敛 | 降级为固定网格点提示（无筛选），仍可作为 baseline |
| 训练不收敛 | 检查学习率、损失函数实现、数据归一化；加入 warmup |
| 算力不足 | 使用 Colab 免费 GPU，减少 epoch 做概念验证 |
