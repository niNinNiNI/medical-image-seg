│  │  │  LoRA Adapter (qkv + proj, r=4)      │    │   │

│  │  └──────────────────────────────────────┘    │   │
│  │  输出: 中间层特征 ×4 + 图像嵌入 (256×64×64)   │   │
│  └───┬──────────────────────────────────────┬───┘   │
│      │                                       │       │
│  ┌───▼──────────────┐              ┌────────▼───┐   │
│  │  Prompt Optimizer │              │  MedDecoder │   │
│  │  32×32网格点采样  │              │  U-Net跳跃  │   │
│  │  逐样本mask解码   │              │  连接解码器  │   │
│  │  输出: SAM mask   │              │  输出: Med   │   │
│  └───┬──────────────┘              │  mask        │   │
│      │                             └────────┬───┘   │
│      └───────────┬──────────────────────────┘       │
│                  │                                   │
│          ┌───────▼───────┐                           │
│          │  可学习融合    │                           │
│          │  α=0.623      │                           │

│          │  + (1-α)·Med  │                           │
│          └───────┬───────┘                           │

### 3.3 SAM-MedSeg 消融实验（5% 数据, seed 42）

| 实验 | Test Dice | IoU | Precision | Recall | 可训练参数 |
|------|:--------:|:---:|:---------:|:------:|:---------:|
| **完整模型** (LoRA + MedDecoder) | 0.448 | 0.320 | 0.421 | 0.697 | 4.84M |

核心结论：**LoRA 是少样本场景的核心驱动力**（仅 227K 参数达最佳泛化），MedDecoder 提供互补的高召回能力。

### 3.6 🎯 核心对比：SAM-MedSeg vs U-Net

| 数据比例 | U-Net | SAM-MedSeg | 优势方 | 差距 |

**交叉点位于 5% → 10% 之间**。SAM-MedSeg 在极端少样本（≤5%）场景下是最优方案。

## 五、已发现并修复的问题

| 问题 | 影响 | 修复方式 |
|------|------|---------|
| SAM mask_decoder 不支持多图 batch | batch_size=2 时 OOM/crash | `full_model.py` 改为逐样本 for-loop 调用 mask_decoder |
| SAM 零样本 NMS CUDA 不兼容 | `SamAutomaticMaskGenerator` 无法在 GPU 运行 | 改用 `sam_zero_shot_baseline.py` 的直接推理方式 |
| 1024×1024 + bs=2 训练 OOM | 8GB 显存不足 | 启用 Gradient Checkpointing（显存从 >8GB → 6.16GB） |
| LoRA 目标模块名不匹配 | `q_proj`/`v_proj` 在 SAM 中不存在 | 改为 `qkv`/`proj`（SAM 使用融合 QKV 投影） |
| SAM 中间层特征形状适配 | B,H,W,C vs B,C,H,W | permute + contiguous 修复 |
| **Checkpoint 加载 key 不匹配** | 后验推理权重未正确加载，Dice 从 0.652 降至 0.396 | `evaluate.py` 和 `visualize_paper_figures.py` 添加 `use_checkpoint=True` |

## 七、论文核心叙事

### 一句话总结

> **SAM-MedSeg 是极端少样本（≤5% 标注数据）医学图像分割的最优方案，交叉点位于 5%-10% 数据区间。**

### 核心论点

1. **零样本 SAM 不可直接用于临床** — Dice 0.271, Precision 0.175
2. **LoRA 是极端少样本的核心驱动力** — 仅 227K 参数在 5% 数据达 0.452 Dice
3. **SAM-MedSeg ≤5% 场景最优** — 1%: +4.0pp, 5%: +2.4pp vs U-Net
4. **交叉点在 5%-10%** — 10% 数据时 U-Net 反超 2.5pp
5. **充足数据下传统方法回归** — U-Net 100%: 0.719 vs SAM-MedSeg: 0.640

### 建议论文结构

1. **Introduction** — 医学图像标注昂贵，少样本分割需求迫切
2. **Related Work** — SAM/MedSAM, LoRA, U-Net, 少样本学习
3. **Method** — SAM Backbone + LoRA + MedDecoder + 自适应融合
4. **Experiments** — 数据效率 + 消融 + 交叉点 + 可视化
5. **Discussion** — 局限（小息肉过度分割）、未来方向（多数据集验证）
6. **Conclusion**
