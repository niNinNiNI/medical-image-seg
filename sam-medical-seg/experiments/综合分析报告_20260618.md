## 二、基线对比：SAM 零样本

### 2.1 实验设置

| 参数 | 值 |
|------|-----|
| 模型 | SAM ViT-B（原始权重，**不做任何微调**） |
| Prompt 策略 | 32×32 = 1024 个网格点（与 SAM-MedSeg 训练一致） |
| 输出 | 单 mask 模式 (`multimask_output=False`) |
| 推理时间 | ~0.31 s/sample（100 张测试集约 31s） |
| 设备 | RTX 4060 Laptop 8GB |

### 2.2 结果

| 指标 | SAM 零样本 | SAM-MedSeg 最佳 (100%) | 提升 |
|------|:---------:|:---------------------:|:----:|
| **Dice** | **0.271** | 0.652 | **+38.1pp** |
| **IoU** | 0.170 | 0.525 | +35.5pp |
| **Precision** | 0.175 | 0.652 | +47.7pp |
| **Recall** | **0.921** | 0.791 | -13.0pp |

### 2.3 关键发现

```
SAM 零样本行为特征：
┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  Recall 0.92 ← 极高！说明 SAM 的 "物体感知" 能力在医学图像   │
│                 上仍然有效，它能找到几乎所有可疑区域。        │
│                                                              │
│  Precision 0.17 ← 极低！SAM 在自然图像上训练的 "分割边界"    │
│                   完全不适用于息肉图像。大量假阳性意味着      │
│                   几乎每个像素都可能被预测为病灶。            │
│                                                              │
│  Dice 0.27 ← 远低于微调模型 0.65，证明了：                   │
│    1. 域差异巨大 — SAM 的自然图像知识与医学图像不兼容         │
│    2. 微调是必需的 — LoRA + MedDecoder 带来 38pp 提升         │
│    3. 零样本 SAM 不能直接用于临床                             │
└──────────────────────────────────────────────────────────────┘
```

**与 SAM-MedSeg 少样本对比**：

| 模型 | 数据量 | Test Dice |
|------|:------:|:---------:|
| SAM 零样本 | 0 | 0.271 |
| SAM-MedSeg 1% | 8 | 0.425 |
| SAM-MedSeg 5% | 40 | 0.467 |
| SAM-MedSeg 10% | 80 | 0.484 |
| SAM-MedSeg 100% | 800 | **0.652** |

> **结论**：即使只用 8 张训练图像（1%），SAM-MedSeg 的 Dice (0.425) 也比零样本 (0.271) 高出 15.4pp，证明了少量标注数据结合参数高效微调的巨大价值。

## 三、基线对比：U-Net

### 3.1 实验设置

| 参数 | 值 |
|------|-----|
| 架构 | 标准 5-level U-Net（Ronneberger et al., 2015） |
| 参数量 | 17.3M（全量训练） |
| 输入/输出 | 3×1024×1024 → 1×1024×1024 |
| 基础通道 | 64（每层翻倍：64→128→256→512→1024） |
| 上采样 | Bilinear interpolation |
| 损失函数 | DiceLoss(0.5) + BCEWithLogitsLoss(0.5) — 与 SAM-MedSeg 一致 |
| 优化器 | AdamW lr=1e-4, CosineAnnealing + 5 epoch warmup |
| 训练 | AMP fp16, batch_size=2, patience=15 |

### 3.2 100% 数据对比

| 指标 | U-Net (seed 42) | SAM-MedSeg 最佳 (seed 44) | U-Net 优势 |
|------|:--------------:|:-------------------------:|:----------:|
| **Test Dice** | **0.723** | 0.652 | **+7.1pp** |
| **IoU** | **0.605** | 0.525 | **+8.0pp** |
| **Precision** | **0.815** | 0.652 | **+16.3pp** |
| **Recall** | 0.734 | 0.791 | -5.7pp |
| **参数量** | **17.3M** | 94.5M | 5.5× 更少 |
| **Best Epoch** | 43 | 29 | U-Net 收敛更慢 |

### 3.3 U-Net 训练轨迹（100%, seed 42）

```
Epoch  Val Dice  阶段
──────────────────────────────────────
  0    0.387    初始
  5    0.662    ★ 突破 0.65（warmup 结束）
 14    0.684    持续上升
 23    0.699    突破 0.69
 28    0.712    突破 0.70
 33    0.717    ★ 首个高点，超越 SAM-MedSeg 最佳 Val Dice (0.705)
 41    0.722    突破 0.72
 43    0.726    👑 最佳！
 58    —        Early stopping 触发（15 epoch 未改进）
```

### 3.4 关键发现：U-Net 为什么在 100% 数据上更优？

```
┌──────────────────────────────────────────────────────────────┐
│ 1. 参数效率更高                                              │
│    U-Net 17.3M vs SAM-MedSeg 94.5M，少 5.5 倍参数。         │
│    在小数据集上，更少的参数 = 更少的过拟合。                  │
│                                                              │
│ 2. 全卷积架构天然适合医学图像                                  │
│    U-Net 的对称编码器-解码器 + 跳跃连接设计与医学图像          │
│    分割任务高度匹配，而 SAM 的 ViT + Prompt 架构最初为        │
│    自然图像交互式分割设计。                                   │
│                                                              │
│ 3. SAM 的 prompt 机制可能不是最优                              │
│    Grid sampling 提示策略在自然图像上有效但在医学图像上        │
│    可能生成次优的提示点，引入噪声。                            │
│                                                              │
│ 4. 精度 vs 召回权衡                                          │
│    U-Net: 高精度(0.815)、中等召回(0.734)                     │
│    SAM-MedSeg: 中等精度(0.652)、高召回(0.791)                │
│    不同的错误偏好，取决于临床需求。                            │
└──────────────────────────────────────────────────────────────┘
```

**⚠️ 重要推论**：
- 在**数据充足**的场景下，U-Net 仍是医学图像分割的更优选择
- SAM-MedSeg 的核心价值在于**少样本场景**（1%-10% 数据）
- 论文叙事应聚焦于：*当标注数据极度稀缺时，预训练知识迁移（SAM + LoRA）的优势*

## 五、消融实验回顾

| 实验 | Test Dice | IoU | Precision | Recall | 可训练参数 |
|------|:--------:|:---:|:---------:|:------:|:---------:|

核心结论：**LoRA 是少样本场景的核心驱动力**（227K 有效参数即达最佳泛化），MedDecoder 提供互补的高召回能力。

## 六、技术总结

### 6.1 成功的技术决策

| 决策 | 效果 | 证据 |
|------|------|------|
| **Gradient Checkpointing** | 使 8GB GPU 能训练 1024×1024 | 显存从 OOM → 6.16 GB |
| **LoRA 低秩适配** | 仅 227K 参数实现有效域适应 | 消融实验：LoRA Only = 0.452 |
| **逐样本 Mask Decoder** | 解决了 SAM 不支持多图 batch 的限制 | CPU/GPU 验证通过 |
| **U-Net 风格跳跃连接** | 保留多尺度空间信息 | MedDecoder 输入 4 个层级特征 |

### 6.2 局限与改进方向

| 局限 | 影响 | 可能改进 |
|------|------|---------|
| **100% 数据不如 U-Net** | U-Net 0.723 > SAM-MedSeg 0.652 | 聚焦少样本场景优势 |
| **Val-Test Gap 仍存** | 100% 数据仍有 4.6pp gap | 更强数据增强、Dropout |
| **Prompt 策略可改进** | Grid sampling 非最优 | 学习式提示或 coarse segmentation |
| **单一数据集** | 泛化性未验证 | ISIC/DDR 等多数据集验证 |
| **SAM 零样本 Precision 极低** | 0.175 的精度无法实用 | —（证明微调必要性） |
