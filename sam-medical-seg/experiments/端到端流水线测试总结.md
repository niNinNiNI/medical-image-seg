## 二、发现并修复的关键问题

### 2.1 LoRA 目标模块名不匹配

**问题**：`LoraConfig(target_modules=["q_proj", "v_proj"])` 无法在 SAM 图像编码器中找到目标模块。

**根因**：SAM ViT-B 使用融合 QKV 投影（`blocks.*.attn.qkv`），而非 HuggingFace Transformers 风格的分立 `q_proj` / `v_proj`。

**修复**：
```python
# 修改前: config.yaml / lora_adapter.py / full_model.py
target_modules: ["q_proj", "v_proj"]

# 修改后
target_modules: ["qkv", "proj"]  # qkv=融合QKV投影, proj=attention输出投影
```

**影响文件**：
- `configs/config.yaml` — 第 64 行
- `models/lora_adapter.py` — 第 54 行
- `models/full_model.py` — 第 91 行

**验证结果**：
```
LoRA Adapter: 89,898,240 total → 227,328 trainable (0.25%)  # qkv+proj
LoRA Adapter: 89,818,368 total → 147,456 trainable (0.16%)  # qkv only
```

### 2.2 SAM 中间层特征形状适配

**问题**：前向传播时 `med_decoder` 收到形状为 `(B, H, W, C)` 的特征，但期望形状为 `(B, C, H, W)`。第一次尝试使用 `permute(0,2,1).reshape(...)` 导致 RuntimeError。

**根因**：SAM 图像编码器的 Transformer block 输出保持空间格式 `(B, H, W, C)`（已验证：12 个 block 全部输出 `[1, 64, 64, 768]`），与我最初假设的序列格式 `(B, N, C)` 不同。

**实验验证**（hook 捕获实际形状）：
```
Block 0:  torch.Size([1, 64, 64, 768])
Block 1:  torch.Size([1, 64, 64, 768])
...
Block 11: torch.Size([1, 64, 64, 768])
Final output: torch.Size([1, 256, 64, 64])
```

**修复**：
```python
# 修改前（错误）
feat = feat.permute(0, 2, 1).reshape(B_val, -1, feat_h, feat_w)

# 修改后（正确）
# (B, H, W, C) → (B, C, H, W)
feat = feat.permute(0, 3, 1, 2).contiguous()
```

**影响文件**：`models/sam_backbone.py` — forward 方法第 205-220 行

**验证结果**（CPU 测试）：
```
MedDecoder input:  [1, 768, 64, 64] × 4 layers ✓
MedDecoder output: [1, 1, 512, 512]             ✓
```

### 2.3 Prompt Encoder API 私有方法调用

**问题**：`prompt_optimizer.py` 调用 `sam_model.prompt_encoder.embed_points()` 导致 AttributeError。

**根因**：SAM 的 PromptEncoder 使用私有方法 `_embed_points()`（下划线前缀），而非公开接口。

**修复**：
```python
# 修改前
sam_model.prompt_encoder.embed_points(chunk_pts)

# 修改后
sam_model.prompt_encoder._embed_points(chunk_pts)
```

**额外优化**：重写了 `prompt_optimizer.py` 整体架构，简化了网格采样逻辑：
- 移除复杂的逐点 SAM 查询（避免逐点调用 mask_decoder 带来的 API 复杂度）
- 新增 `center` 策略（单点中心提示，最快）
- 新增 `gt_guided` 策略（训练时从 GT mask 采样正负点，oracle 模式）
- 更新 `grid_sampling` 策略（固定角点 + 随机网格点组合）

### 2.4 GPU 显存不足（OOM）→ 已通过 Gradient Checkpointing 解决

**问题**：RTX 4060 8GB 在 1024×1024 分辨率 + batch_size=2 时 OOM。

**根因分析**：SAM ViT-B 在 1024 分辨率下产生 64×64=4096 个 patch token。每层 attention map 大小为 `(batch, 12 heads, 4096, 4096)` ≈ 768 MB（fp32）/ 384 MB（fp16）。12 层 transformer 的激活存储远超 8GB。

**初始缓解方案**（2026-06-14）：降低分辨率和 batch size
```yaml
# configs/config.yaml
data:
  image_size: [512, 512]        # 从 1024 降为 512
training:
  batch_size: 1                 # 从 2 降为 1
  mixed_precision: true         # AMP fp16 保持开启
```

**配套修复**：修改 `data/dataset.py` 支持从 1024 预处理的 .npz 文件动态 resize 到 512：
```python
# 加载 .npz 后自动检查尺寸并 resize
if (h, w) != self.image_size:
    image = F.interpolate(image.unsqueeze(0), size=self.image_size, mode="bilinear")
    mask = F.interpolate(mask.unsqueeze(0), size=self.image_size, mode="nearest")
```

**512×512 显存占用**：
```
SAM ViT-B 编码器:      ~2.0 GB
LoRA 梯度 (qkv):       ~0.1 GB
Mask Decoder:          ~0.3 GB
Med Decoder:           ~0.1 GB
AMP 临时张量:          ~0.5 GB
总计:                  ~3.0 GB ✅ (8GB 余量充足)
```

**最终方案**（2026-06-15）：通过 Gradient Checkpointing + AMP 恢复到 1024×1024 + batch_size=2。详见 2.6 和 2.7 节。

### 2.6 SAM Mask Decoder 多图 batch 不兼容（2026-06-15 新发现）

**问题**：batch_size=2 时，CPU 端到端测试在 `mask_decoder.predict_masks` 中抛出 `RuntimeError: The size of tensor a (4) must match the size of tensor b (2) at non-singleton dimension 0`。

**根因**：SAM 的 `mask_decoder` 设计为**单图多 prompt** 模式，而非**多图 batch** 模式。在 `predict_masks` 中（`mask_decoder.py:126-128`）：

```python
# tokens.shape[0] = B（sparse_prompt_embeddings 的 batch 维度）
tokens = torch.cat((output_tokens, sparse_prompt_embeddings), dim=1)

# 将 image_embeddings 按 tokens 数量 repeat — 这里把 B 当成了"每张图的 prompt 组数"
src = torch.repeat_interleave(image_embeddings, tokens.shape[0], dim=0)
# image_embeddings (B=2, 256, 64, 64) → src (B*B=4, 256, 64, 64)
# dense_prompt_embeddings (B=2, 256, 64, 64)
# src + dense_prompt_embeddings → 维度不匹配!
```

SAM 原本的用法是：1 张图 + N 组 prompt → batch=N。当我们传入 batch=B 张图时，`repeat_interleave(image_embeddings, B)` 会将每张图膨胀 B 倍，导致维度不匹配。

**修复**（`models/full_model.py` — forward 方法第 173-195 行）：
```python
# 修改前（批量调用 — 不兼容）
sparse_embeddings, dense_embeddings = self.backbone.sam.prompt_encoder(
    points=(points, labels), boxes=None, masks=None)
sam_masks, _ = self.backbone.sam.mask_decoder(
    image_embeddings=image_embeddings, ...)

# 修改后（逐样本处理）
image_pe = self.backbone.sam.prompt_encoder.get_dense_pe()
sam_masks_list = []
for b_idx in range(B):
    single_pts = points[b_idx:b_idx+1]
    single_labels = labels[b_idx:b_idx+1]
    sparse_emb, dense_emb = self.backbone.sam.prompt_encoder(
        points=(single_pts, single_labels), boxes=None, masks=None)
    single_mask, _ = self.backbone.sam.mask_decoder(
        image_embeddings=image_embeddings[b_idx:b_idx+1],
        image_pe=image_pe,
        sparse_prompt_embeddings=sparse_emb,
        dense_prompt_embeddings=dense_emb,
        multimask_output=False)
    sam_masks_list.append(single_mask)
sam_masks = torch.cat(sam_masks_list, dim=0)
```

**验证结果**（CPU 端到端测试，1024×1024，batch=2）：
```
Output shape: torch.Size([2, 1, 1024, 1024])   ✅
Loss: 0.8653                                    ✅
  dice: 0.3817                                  ✅
  iou: 0.2362                                   ✅
```

### 2.7 Gradient Checkpointing — 1024×1024 训练的关键技术（2026-06-15 新增）

**问题**：即使修复了 mask_decoder batch 兼容性，1024×1024 分辨率在训练模式下仍因显存不足 OOM（forward no_grad 可通过，但 backward 需要存储中间激活）。

**根因**：SAM ViT-B 的 12 层 transformer 在 backward 时需保留中间激活。关键瓶颈在 `add_decomposed_rel_pos`（`image_encoder.py:358`）：
```python
attn.view(B, q_h, q_w, k_h, k_w) + rel_h[:, :, :, :, None] + rel_w[:, :, :, None, :]
# (B, 12, 64, 64, 64, 64) → 单层 ~768 MB (fp32) / ~384 MB (fp16)
# 12 层累计 → 远超 8GB
```

**解决方案**：Gradient Checkpointing — 以时间换空间，前向时不存储中间激活，backward 时重新计算。

**实现**（`models/sam_backbone.py` — 新增方法）：
```python
def enable_gradient_checkpointing(self) -> None:
    import torch.utils.checkpoint as cp

    class CheckpointBlock(nn.Module):
        def __init__(self, block):
            super().__init__()
            self.block = block

        def forward(self, x):
            return cp.checkpoint(self.block, x, use_reentrant=False)

    blocks = getattr(self.image_encoder, "blocks", None)
    if blocks is not None:
        for i, block in enumerate(blocks):
            if not isinstance(block, CheckpointBlock):
                blocks[i] = CheckpointBlock(block)
```

**配套修改**：
- `models/full_model.py`：新增 `use_checkpoint` 参数，在 LoRA 应用后调用 `self.backbone.enable_gradient_checkpointing()`
- `configs/config.yaml`：新增 `training.gradient_checkpointing: true`
- `train.py`：从 config 读取 `gradient_checkpointing` 并传入 `SAMMedSeg()`

**显存对比（1024×1024，AMP + batch=2）**：

| 配置 | 不启用 Checkpoint | 启用 Checkpoint |
|------|-------------------|-----------------|
| 推理 (no_grad) | 5.20 GB | 1.59 GB |
| 训练 (forward+backward+step) | OOM (>8 GB) | **6.16 GB** ✅ |
| 训练 (batch=1) | OOM (>8 GB) | **3.28 GB** ✅ |

**性能代价**：约 30% 额外前向计算，换取 ~60% 激活显存节省。对于 8GB GPU 而言是必要的权衡。

## 六、已知局限与注意事项

### 6.1 显存限制
- **1024×1024 + batch=1**：AMP + gradient checkpointing → ~3.28 GB ✅
- **1024×1024 + batch=2**：AMP + gradient checkpointing → ~6.16 GB ✅（8GB 可用）
- **推理模式**：无需 checkpointing，1024×1024 + batch=2 → ~5.20 GB
- **降级方案**：若需更低显存，可降至 512×512 + batch=1 → ~3.0 GB

### 6.2 Gradient Checkpointing 性能权衡
- 启用后前向计算量增加约 30%（backward 时需重新计算中间激活）
- 换取约 60% 的激活显存节省，是 8GB GPU 运行 1024 训练的关键
- 推理时不需要，可在推理前通过 `torch.utils.checkpoint` 的切换关闭

### 6.3 SAM Mask Decoder batch 限制
- SAM 原生 `mask_decoder` 不支持多图 batch，当前通过逐样本 for-loop 处理
- 对训练速度有轻微影响（串行化 mask_decoder 调用），但 mask_decoder 本身轻量（~0.3GB）
- 主要瓶颈仍在 image_encoder，不受此限制影响

### 6.4 PromptOptimizer 简化
- 当前 `grid_sampling` 策略使用固定角点 + 随机采样，而非完整的 IoU 筛选
- 完整的逐点查询 + IoU 筛选 + NMS 实现由于 SAM API 复杂性暂未集成
- 训练时优先使用 `gt_guided` 策略（从 GT 采样 oracle 点），推理时使用 `center` 或 `grid_sampling`

### 6.5 SAM 固定输入分辨率
- SAM 图像编码器要求严格 1024×1024 输入（位置编码固定为 64×64）
- 当前在 `sam_backbone.py` 中自动 padding 到 1024，再在 MedDecoder 输出 resize 到目标尺寸
- 这意味着编码器始终在 1024 分辨率计算，`image_size` 参数仅影响 MedDecoder 输出层

### 6.6 数据预处理
- .npz 文件以 1024×1024 存储，训练时通过 `dataset.py` 动态 resize 到目标尺寸
- 切换训练分辨率只需修改 `config.yaml` 中 `data.image_size` 和 `training.gradient_checkpointing`
